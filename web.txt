How JavaScript Engines Interpret and Execute Code?
Parsing: When we load a webpage or execute a script, the JavaScript engine first parses the source code to understand its structure. It converts the code into an Abstract Syntax Tree (AST) which is a hierarchical representation of the script.
Compilation: Now in compilation phase, the engine translates the AST into machine-readable bytecode using JIT (Just-In-Time) compilation. JIT compilation optimizes performance by compiling frequently executed code segments at runtime.
Execution: Finally, the bytecode or machine code is executed line by line, which produces the output or behavior as defined by the JavaScript code.

Stopping Delegation / BubblingMethod: event.stopPropagation()Action: Stops the event from traveling further up (or down in the capturing phase) the DOM tree.Use Case: Perfect for event delegation or nested elements. For example, if you have a button inside a parent <div> and you click the button, stopPropagation() prevents the click event from reaching and triggering the parent <div>'s click listener.2. Immediate StoppingMethod: event.stopImmediatePropagation()Action: Does everything stopPropagation() does, but also prevents any other event listeners attached to the same current element from executing.Use Case: Use this when an element has multiple event listeners for the same event type (e.g., two click listeners), and you want to ensure the current handler is the last one to run.3. Default Stopping (Bonus)Method: event.preventDefault()Action: Stops the browser's default behavior associated with an event (like submitting a form or following a link).Note: This does not stop bubbling. To block both the default action and the bubbling, you must chain both methods inside your function.
